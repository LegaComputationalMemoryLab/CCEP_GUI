# Coordinate file format

The analyzer expects a comma-separated values file generated by the local electrode-localization workflow. The file is not intended to contain raw imaging data.

## Required columns

| Column | Required | Units | Description |
|---|---:|---|---|
| `ElecNumber` | Yes | integer | Electrode/contact order. The current loader assumes this order can be aligned with EDF channel order. |
| `X` | Yes | millimeters | x-coordinate for the contact. |
| `Y` | Yes | millimeters | y-coordinate for the contact. |
| `Z` | Yes | millimeters | z-coordinate for the contact. |
| `Regions` | No | text | Anatomical label used for ROI filtering. Missing values are set to `Unknown`. |

## Recommended columns for future versions

| Column | Description |
|---|---|
| `ContactLabel` | Human-readable label matching clinical channel names. |
| `Hemisphere` | `L`, `R`, or `Midline`. |
| `InBrain` | Boolean flag for excluding extracerebral contacts from ROI summaries. |
| `LocalizationMethod` | Workflow used to generate coordinates. |
| `CoordinateFrame` | Native, MNI, ACPC, or other frame. |

## Privacy warning

Clinical coordinate files can be identifiable when paired with electrode labels, implantation pattern, imaging metadata, dates, or subject identifiers. Do not commit patient coordinate files to GitHub.
